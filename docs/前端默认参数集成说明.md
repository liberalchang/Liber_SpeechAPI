# 前端默认参数集成说明

## 概述

本文档说明了如何将前端默认参数从硬编码改为从后端动态加载的实现方案。

## 实现内容

### 1. 后端默认参数接口

**位置**: `src/liber_speech/api/routes.py`

**接口**: `GET /api/v1/default-params`

**功能**: 返回完整的默认参数配置，包括 ASR、TTS、API 和全局配置。

```python
@router.get("/default-params")
async def default_params(
    _: Annotated[str | None, Depends(get_auth_credentials)],
):
    """获取后端默认参数。"""
    return load_default_params()
```

### 2. 前端API客户端扩展

**位置**: `web/src/lib/api.ts`

**新增方法**: `getDefaultParams()`

```typescript
async getDefaultParams(): Promise<DefaultParams | ErrorResponse> {
    console.log("前端: 发送获取默认参数请求");
    const res = await fetch(this.getFullUrl("/default-params"), {
        headers: this.headers(false),
    });
    const result = await res.json();
    console.log("前端: 默认参数响应", result);
    return result;
}
```

### 3. 前端页面集成

#### ASR页面 (`web/src/pages/AsrPage.tsx`)

- 添加 `useEffect` 钩子在页面加载时获取默认参数
- 自动更新表单字段的默认值
- 支持所有ASR相关参数的动态加载

#### TTS页面 (`web/src/pages/TtsPage.tsx`)

- 添加 `useEffect` 钩子在页面加载时获取默认参数
- 自动更新表单字段的默认值
- 支持所有TTS相关参数的动态加载

## 使用方式

### 1. 启动后端服务

```bash
# 使用uv环境
uv run python -m liber_speech.api

# 或使用传统方式
python -m liber_speech.api
```

### 2. 启动前端服务

```bash
cd web
npm run dev
```

### 3. 访问页面

- ASR页面: http://localhost:5173/asr
- TTS页面: http://localhost:5173/tts

页面加载时会自动从后端获取默认参数并填充到表单中。

## 配置文件

默认参数可以通过以下方式配置：

1. **环境变量** (优先级最高)
   - `LIBER_ASR_MODEL`: ASR模型
   - `LIBER_TTS_MODEL`: TTS模型
   - `LIBER_DEVICE`: 推理设备
   - `LIBER_RESULTS_DIR`: 结果目录

2. **配置文件** `src/liber_speech/config/default_params.json`

3. **代码中的备用默认值** (优先级最低)

## 测试

运行测试脚本验证默认参数加载：

```bash
python test_default_params_simple.py
```

## 优势

1. **集中管理**: 所有默认参数在后端统一管理
2. **动态配置**: 可以通过环境变量或配置文件动态调整
3. **一致性**: 前后端参数保持同步
4. **可维护性**: 修改默认值只需在后端进行

## 注意事项

1. 前端页面加载时需要后端服务已经启动
2. 如果获取默认参数失败，前端会使用硬编码的备用值
3. 认证: 默认参数接口需要API认证（如果启用了认证模式）
