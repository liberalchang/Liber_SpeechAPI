## 2026-03-27 初始化与 M1（核心引擎）

### 目标
- 搭建项目结构与依赖管理（`uv`）
- 实现核心引擎层（ASR/TTS 封装、设备选择、缓存、统一 I/O 与错误处理）
- 支持同步推理

### 关键文件与功能

#### `pyproject.toml`
- 定义项目信息与依赖（`pydantic`、`typer`、`rich`、`transformers`、`torch`、`chatterbox-tts` 等）
- 配置 `hatchling` 构建系统，使用 `src` 布局
- CLI 入口 `liber-speech` 指向 `liber_speech.cli.app:app`

#### `src/liber_speech/engines/`
- `device.py`：设备选择逻辑（`cpu`/`cuda:N`），自动检测 CUDA
- `io_types.py`：ASR/TTS 的 Pydantic 数据结构（`ASRResult`、`TTSResult`）
- `model_cache.py`：HuggingFace 缓存目录解析（支持 `LIBER_HF_CACHE_DIR` 环境变量）
- `errors.py`：统一异常基类（`LiberSpeechError`、`ConfigError`、`DependencyError`、`InferenceError`）
- `asr_engine.py`：封装 `transformers.pipeline` 实现 Whisper ASR，支持 `transcribe`/`translate`、语言、时间戳
- `tts_engine.py`：封装 `chatterbox-tts` 实现 TTS，支持 `turbo`/`standard`/`multilingual`、音色克隆

#### `src/liber_speech/cli/`
- `app.py`：CLI 主入口，使用 Typer，注册子命令模块
- `__main__.py`：支持 `python -m liber_speech.cli` 运行

### 验证命令（计划）
```bash
uv sync
uv run liber-speech version
uv run python -c "from liber_speech.engines import ASREngine, TTSEngine; print('engines ok')"
```

## 2026-03-31 M2（CLI）- ASR/TTS/Doctor 子命令

### 目标
- 完善 CLI 子命令：`asr transcribe/translate`、`tts synthesize`、`doctor`
- 支持多种输出格式（ASR：JSON/TXT/SRT/VTT；TTS：WAV/MP4/OGG_OPUS）
- 提供环境诊断功能（Python、PyTorch、CUDA、FFmpeg、HF 缓存）

### 新增文件与功能

#### `src/liber_speech/cli/commands_asr.py`
- `asr transcribe`：转录音频，支持 `--model`、`--device`、`--language`、`--timestamps`、`--out`，根据后缀自动保存为 JSON/TXT/SRT/VTT
- `asr translate`：翻译音频（task=translate），参数同 `transcribe`

#### `src/liber_speech/cli/commands_tts.py`
- `tts synthesize`：合成语音，支持 `--model`（turbo/standard/multilingual）、`--device`、`--language`、`--audio-prompt`（音色克隆）、`--format`（wav/mp4/ogg_opus）
- 使用 `torchaudio.save` 保存 WAV，使用 `ffmpeg` 转码为 MP4/AAC 与 OGG/Opus（按 Telegram Voice Note 最佳实践：单声道 48kHz、28kbps VBR、voip 应用、响度标准化 -16 LUFS）

#### `src/liber_speech/cli/commands_doctor.py`
- `doctor`：诊断环境，输出 Python、PyTorch（含 CUDA）、Transformers、Chatterbox、FFmpeg、HF 缓存状态
- Windows 上给出 CUDA/Torch 兼容性提示

#### `src/liber_speech/engines/io_utils.py`
- 统一 I/O 工具函数：`save_wav`、`save_mp4`、`save_ogg_opus`、`save_asr_json/txt/srt/vtt`、`convert_with_ffmpeg`
- 支持自动创建父目录；OGG/Opus 按 Telegram Voice Note 最佳实践转码

#### `src/liber_speech/cli/commands_warmup.py`
- `warmup`：预加载 ASR 与 TTS 模型到内存，减少首次推理延迟；支持 `--asr-model`、`--tts-model`、`--device`

#### CLI 主入口更新
- 在 `app.py` 中注册 `asr`、`tts`、`doctor`、`warmup` 四个子命令模块

### 典型使用命令（计划）
```bash
# ASR 转录（自动检测设备，输出 JSON）
uv run liber-speech asr transcribe audio.wav

# ASR 翻译（指定模型与语言，输出 SRT）
uv run liber-speech asr translate audio.wav --out subs.srt --language zh --model openai/whisper-large-v3

# TTS 合成（多语模型，输出 OGG/Opus）
uv run liber-speech tts synthesize "你好世界" --out hello.ogg --format ogg_opus --language zh

# 诊断环境
uv run liber-speech doctor

# 模型预暖
uv run liber-speech warmup --device cuda:0 --asr-model openai/whisper-large-v3 --tts-model multilingual
```

## 2026-03-31 M3（FastAPI 服务）

### 目标
- 实现 API 服务：`serve` 命令
- 支持 API Key 鉴权（`Authorization: Bearer <KEY>`）
- 同步模式（默认）与异步任务 + 下载接口
- 健康检查与预热端点

### 新增文件与功能

#### `src/liber_speech/api/`
- `app.py`：FastAPI 应用工厂，全局异常处理，CORS，生命周期
- `config.py`：从环境变量加载配置（鉴权模式、API Keys、结果目录、TTL）
- `deps.py`：鉴权依赖（支持 `none`/`api_key` 模式）
- `models.py`：Pydantic 请求/响应模型（ASR/TTS 同步、异步任务、健康检查）
- `routes.py`：API 路由实现（健康检查、预热、ASR 同步、TTS 同步、异步任务框架、结果下载）

#### `src/liber_speech/cli/commands_serve.py`
- `serve`：启动 FastAPI 服务，支持 `--host`/`--port`

#### `pyproject.toml`
- 添加 `[project.scripts]` 确保 CLI 入口正确打包
- API 依赖放入 `[project.optional-dependencies]`（`fastapi`、`uvicorn[standard]`）

### 环境变量配置
- `LIBER_API_AUTH_MODE`：鉴权模式（none/api_key）
- `LIBER_API_KEYS`：逗号分隔的 API Keys
- `LIBER_API_HOST`/`LIBER_API_PORT`：监听地址与端口
- `LIBER_RESULTS_DIR`：异步任务结果目录
- `LIBER_RESULTS_TTL_HOURS`：结果保留时长（默认 24h）

### API 端点（计划）
- `GET /api/v1/health`：健康检查
- `POST /api/v1/warmup`：预热模型
- `POST /api/v1/asr/transcribe`：ASR 转录（同步，支持文件上传/URL）
- `POST /api/v1/tts/synthesize`：TTS 合成（同步，支持音色克隆、多格式）
- `POST /api/v1/tasks/asr/transcribe`：提交 ASR 异步任务（框架）
- `GET /api/v1/tasks/{task_id}`：查询任务状态（框架）
- `GET /api/v1/results/{filename}`：下载结果文件

### 典型使用命令（计划）
```bash
# 安装 API 依赖
uv sync --extra api

# 启动服务（默认 8000）
uv run liber-speech serve

# 启动服务（指定端口）
uv run liber-speech serve --host 0.0.0.0 --port 8080

# API 调用示例（需配置 API Key）
curl -X POST "http://localhost:8000/api/v1/tts/synthesize" \
  -H "Authorization: Bearer YOUR_KEY" \
  -F text="你好世界" \
  -F format="ogg_opus" \
  -F language="zh"
```

## 2026-03-31 M4（交互式安装脚本）

### 目标
- 跨平台交互式安装脚本（Windows PowerShell / Unix Shell）
- 组件选择（ASR/TTS/API）
- 设备选择（CPU/CUDA）
- 默认模型配置
- 生成 `.env` 配置文件
- CLI 命令默认值读取环境配置

### 新增文件与功能

#### `setup.ps1`（Windows PowerShell）
- 交互式选择组件、设备、模型、鉴权模式、结果目录、HF 缓存
- 生成 `.env` 配置文件
- 自动执行 `uv sync [--extra api]`
- 验证 CLI 入口

#### `setup.sh`（Linux/macOS）
- 与 PowerShell 脚本功能对等
- 颜色输出与错误处理
- 检测 `uv` 命令是否存在

#### `src/liber_speech/settings.py`
- 定义 CLI 配置模型（`CLIConfig`、`ASRConfig`、`TTSConfig`）
- 从环境变量加载默认配置（`load_cli_config`）
- 支持设备、模型、语言、格式等默认值

#### CLI 命令更新
- `commands_asr.py` 与 `commands_tts.py` 从 `settings.py` 读取默认模型与设备
- 参数默认值改为空字符串，内部优先使用环境配置

### 环境变量（由安装脚本生成）
- `LIBER_DEVICE`：默认推理设备
- `LIBER_ASR_MODEL`：默认 ASR 模型
- `LIBER_TTS_MODEL`：默认 TTS 模型
- `LIBER_API_AUTH_MODE`/`LIBER_API_KEYS`：API 鉴权
- `LIBER_API_HOST`/`LIBER_API_PORT`：监听地址与端口
- `LIBER_RESULTS_DIR`/`LIBER_RESULTS_TTL_HOURS`：异步任务结果管理
- `LIBER_HF_CACHE_DIR`：HuggingFace 缓存目录

### 典型使用流程
```bash
# Windows
.\setup.ps1

# Linux/macOS
./setup.sh

# 安装后，CLI 命令将使用环境默认值
uv run liber-speech asr transcribe audio.wav  # 使用默认模型/设备
uv run liber-speech tts synthesize "你好" --format ogg_opus  # 使用默认模型/设备
uv run liber-speech serve  # 使用环境配置的鉴权与端口
```

## 总结

至此，Liber Speech API 已完成四个里程碑：

- **M1（核心引擎）**：ASR/TTS 封装、设备选择、缓存、统一 I/O
- **M2（CLI）**：`asr`/`tts`/`doctor`/`warmup` 子命令，多格式输出
- **M3（FastAPI 服务）**：API 服务、鉴权、同步/异步框架、结果下载
- **M4（交互式安装）**：跨平台安装脚本、环境配置生成、CLI 默认值

项目已具备本地 CLI 与远程 API 双模式，支持多平台部署与交互式配置，可进入测试与文档完善阶段。
