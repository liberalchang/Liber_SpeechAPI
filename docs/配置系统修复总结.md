# 配置系统修复总结

## 问题分析

用户报告了以下问题：
1. **GPU模式未生效** - 配置了 `LIBER_DEVICE=cuda` 但系统使用CPU
2. **输出路径未生效** - 配置了 `LIBER_RESULTS_DIR=results` 但文件保存在执行目录
3. **参数名称问题** - 为什么还有 `--out` 参数而不是 `--output`

## 根本原因

### 1. 环境变量未加载
- **问题**: 启动脚本没有加载 `.env` 文件
- **原因**: 使用 `python -S` 跳过site模块，导致 `python-dotenv` 无法正常工作
- **解决**: 在 `scripts/run_cli.py` 中手动解析 `.env` 文件

### 2. 参数优先级错误
- **问题**: 参数管理器的默认值覆盖了环境变量
- **原因**: `default_params.json` 缓存文件包含旧的 `"device": "auto"` 配置
- **解决**: 修改 `default_params.py` 直接从环境变量读取，删除缓存文件

### 3. 输出路径逻辑缺失
- **问题**: TTS命令没有使用 `LIBER_RESULTS_DIR` 环境变量
- **原因**: CLI参数处理逻辑中缺少输出路径的环境变量集成
- **解决**: 在 `commands_tts.py` 中添加输出路径处理逻辑

### 4. 设备选择过于严格
- **问题**: 配置CUDA但不可用时直接报错
- **原因**: `device.py` 中的设备选择逻辑没有回退机制
- **解决**: 添加自动回退到CPU的逻辑并给出警告

## 配置说明

### 环境变量优先级
1. **用户命令行参数** (最高优先级)
2. **环境变量** (.env 文件)
3. **默认值** (fallback defaults)

### 输出路径逻辑
- 如果用户指定 `--out`/`--output` 参数：使用指定路径
- 如果用户未指定：使用 `LIBER_RESULTS_DIR` 环境变量 + 默认文件名
- 如果 `LIBER_RESULTS_DIR` 未设置：默认使用 `results` 目录

### 设备选择逻辑
- 如果配置CUDA但不可用：自动回退到CPU并警告
- 如果配置CPU：直接使用CPU
- 如果配置auto：优先CUDA，否则CPU
